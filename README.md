# Прогнозування цін автомобілів

Мета цього проєкту — передбачити ціни автомобілів на основі різних характеристик, таких як модель, рік випуску, пробіг, тип палива тощо.

## Дані

Датасет містить числові та категоріальні ознаки автомобілів. Цільова змінна — **ціна автомобіля**.  

Категоріальні ознаки потрібно закодувати для використання в регресійній моделі.

Було перевірено два підходи кодування:

1. **One-Hot Encoding**  
2. **LabelEncoder**

1. Спочатку модель оцінювалася через **train_test_split**.  
2. Потім для більш надійної перевірки використано **cross_val_score**. 

Обидва методи кодування показали **схожі результати** за метриками якості.  

```log
INFO:__main__:one-hot:
INFO:root:MAE (mean): 24134.721938111114
INFO:root:MSE (mean): 784815185.4721903
INFO:__main__:label-encoder:
INFO:root:MAE (mean): 24155.22318991111
INFO:root:MSE (mean): 787741567.70698
```

У подальшому коді буде використовуватися LabelEncoder через те, що він займає менше місця, при цьому не погіршує точність моделі, а також спрощує роботу з великими датасетами, зменшуючи кількість колонок і обсяг пам’яті, необхідний для обробки категоріальних ознак.
